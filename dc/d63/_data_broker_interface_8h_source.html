21 /*
22  *
23  *
24  * HOW TO:
25  * 1) producer:
26  * - push an initial data set and get an ID back
27  * - then call pushData(ID, ...)
28  * 2) receiver:
29  * a) Asynchronous (this should be the default):
30  * - call registerAsyncReceiver() with (sensor group and name)
31  * - updates the receiver from another thread than the pushing thread.
32  * NOTE: If the producer pushes fast you only receive the last datum.
33  * b) Synchronous
34  * - call registerSyncReceiver() with (sensor group and name)
35  * - updates the receiver from within the pushData-call.
36  * NOTE: only use this if it is important that you don't miss a datum.
37  * c) Timed:
38  * - call createTimer() with a suitable name
39  * - call registerTimedReceiver() with a desired update period
40  * - call stepTimer() to step the timer and get the updates
41  * NOTE: you will get the latest datum. regardless whether it's been
42  * updated or not since the last update.
43  * d) Triggered:
44  * - call createTrigger() with a suitable triggerName
45  * - call registerTriggered() to register to a certain trigger
46  * - call trigger() to manually trigger a certain trigger and update
47  * all receivers registered to it.
48  * NOTE: you will get the latest datum. regardless whether it's been
49  * updated or not since the last update.
50  *
51  * TYPICAL USECASES:
52  * 1) Plotter:
53  * should run in Asynchronous or Timed mode
54  * 2) File Logger:
55  * should run in Asynchronous or Timed mode
56  * 3) Controller:
57  * should probably run in Asynchronous mode.
58  * 4) Command:
59  * When listening for commands you should run in Synchronous mode.
60  */
