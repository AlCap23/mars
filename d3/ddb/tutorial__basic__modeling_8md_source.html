1 Basic Modeling {#tutorial_basic_modeling}
2 ==============
3 
4 > NOTE: This tutorial refers to creating the deprecated MARS Scene format. MARS will remain backwards compatible to MARS scenes as long as reasonably possible, but the SMURF format primarily supported in future development is SMURF (refer to [Supported 3D Formats](@ref supported_3d_formats)).
5 >
6 > To use Blender for the creation of SMURF, check out the Blender Add-On [Phobos](http://www.github.com/rock-simulation/phobos).
7 
8 ## Introduction
9 
10 In this tutorial you will learn how to build a simple model of a robot
11 in Blender and how to export it to MARS.
12 
13 ## Prerequisites
14 
15 We assume you already have a MARS development environment setup in
16 "~/devel/". You should also have a recent version of Blender
17 installed (we use 2.62 in this tutorial).
18 
19 ## Blender Basics
20 
21 Blender's shortcuts and hotkeys depend on the context in which they
22 are triggered. The context is usually determined by the mouse cursor.
23 That means that a certain hotkey, "alt+p" for example, will have
24 different meanings depending on the position of the mouse. When the
25 mouse is hovering the 3D-View the hotkey "alt+p" will break up
26 child-parent relations, when hovering the text editor it will run the
27 current script.
28 
29 ### 3D-View Shortcuts and Hotkeys
30 
31 | Operation | Shortcut |
32 | ----------- | -------- |
33 | rotate view | middle mouse button |
34 | move view | shift + middle mouse button |
35 | zoom view | mouse wheel or ctrl + middle mouse button |
36 | move object (grab) | g |
37 | rotate object | r |
38 | scale object | s |

45 ### Rescaling Objects
46 
47 
48 If you edit the dimensions of an object in "object mode" you might
49 notice that the scale changes automatically. This is because in
50 object mode you do not really change the object's vertices. To apply
51 the changes you did in "object mode" to the object's vertices you can
52 press "ctrl+a" (with the mouse hovering over the 3D-View) and choose
53 scale. You will notice that the scale in "object mode" is back to (1,
54 1, 1). Alternatively you can select this option from the menu
55 object->apply->scale

59 ## Building the Model
60 
61 1. Start by creating a box of the dimensions 0.5 x 0.3 x 0.2 this can
62  be done by directly editing the object properties "Dimensions" in
63  the toolbar.
64 
65 2. Rename the object (e.g. "Body") this can be done in the object
66  property window.
67 
68  ![Renaming an object](images/tutorials/basic_modeling/rename_object2.png)
69 
70 3. Choose a color by selecting a diffuse and/or specular color in the
71  material property window.
72 
73  ![Material properties](images/tutorials/basic_modeling/material.png)
74 
75 4. It is convenient to have the model on a different layer than the
76  camera and lights. Therefore we move it to a different layer by
77  selecting everything belonging to the model (so far only the box).
78  Then press "m" and select a new layer to move the objects to that
79  layer.
80 
81 5. Create a sphere that will serve as a wheel. In the Menu select
82  "Add->Mesh->UV Sphere" and name the object (e.g., "Wheel.000").
83 
84  ![Create Sphere](images/tutorials/basic_modeling/create_sphere2.png)
85 
86 6. Scale the sphere to an appropriate size (we used 0.1 in all
87  dimensions)
88 
89 7. Position the sphere at a lower corner of the box be setting its
90  location parameters (e.g., x=0.25; y=-0.15; z=-0.1)
91 
92 8. Create a new material by pressing "new" in the material property
93  window
94 
95  ![New Material](images/tutorials/basic_modeling/new_material.png)
96 
97 9. Set material properties for the wheel/sphere.
98 
99 10. Set up parent-child relationship between the body and the wheel.
100  Select the child and the parent (in that order) and press
101  "ctrl+p". NOTE: If you want to break up a parent-child
102  relationship you should select the child object in the 3D-View and
103  press "alt+p" (while hovering the 3D-View) and select "Clear and
104  Keep Transformation".
105 
106 11. The way we currently export joints from blender to MARS is via
107  helper objects that contain all relevant information (e.g., joint
108  and motor type, limits, etc.) in custom properties. So go ahead
109  and create a arbitrary object (we usually use slim cylinders
110  because you can clearly see the rotation axis) and position it at
111  the location of the joint. It helps if the child object of the
112  joint has the same origin and orientation as the joint helper
113  object. Create two custom properties in the "object property"
114  window.
115 
116  ![Joint](images/tutorials/basic_modeling/joint.png)
117 
118  For the first custom property press edit to set the "property
119  name" to "type" and the "property value" to "joint" for the second
120  custom property press edit to set the "property name" to "node2"
121  and the "property value" to "Wheel.000" this tells the export
122  script that this is a helper object for a "joint" and that the
123  object on the output shaft is the "Wheel.000" object. The object
124  on the input shaft it determined by the parent of the helper
125  object itself so set that to "Body".
126 
127 12. By default each joint is automatically associated with a servo
128  motor. To change the motor type to a PID motor you need to create
129  another custom property called "motor_type" and set its value to
130  "2" (note that you might need to increase the "Max" value in the
131  custom property)
132 
133 13. Turn the timeline into a text editor
134 
135  ![Text Editor](images/tutorials/basic_modeling/text_editor.png)
136 
137 14. By selecting the menu "text->Open Text Block" in the text editor
138  open the "create_mars_props.py" script from
139  ~/devel/mars/scripts/blender/
140 
141 15. Select all objects of the model in the 3d-view and run the script
142  by pressing "Run Script" in the text editor window. You will see
143  that this creates various custom properties on every object. You
144  are free to edit these properties (e.g. set the mass)
145 
146 16. For the wheel create two more custom properties: "physicMode" and
147  "radius". Set "physicMode" to "sphere" and "radius" to half the
148  diameter (in our case 0.05). Otherwise the physical
149  representation of the wheels in the simulation would be boxes
150  makeing very poor wheels.
151 
152 17. Duplicate the wheel and the helper object by selecting them and
153  pressing "shift+d". A duplicated object will inherit the
154  location, orientation, material, parent, and custom properties
155  among other things. By nameing the first wheel "Wheel.000" we
156  ensure consistent nameing. Blender automatically increases the
157  ending to "Wheel.001" for the duplication. Of course you can set
158  the name to what ever you like.
159 
160 18. Also create the last two remaining wheels by duplicating one of
161  the first. Position the four wheels at the four lower corners of
162  the box.
163 
164 19. Do not forget to change the value of the "node2" in each of the
165  copied helper objects' custom property.
166 
167 20. Add the custom properties "filename" and "path" to the World and
168  fill in where you want to export the scene to. Ideally, this
169  should be an empty directory because bside the .scn file the .obj
170  and .scene files are also exported for debuging purposes.
171 
172  ![World Properties](images/tutorials/basic_modeling/world_properties.png)
173 
174 21. Load the "relative_mars_export.py" from
175  ~/devel/mars/scripts/blender/ in the text editor window.
176 
177 22. Select all objects of the model and run the relative_mars_export
178  script.
179 
180 23. Do not forget to save your blender scene.
181 
182 Congratulations! You have your first blender created MARS scene.
183 If the scene is not correctly displayed in MARS (e.g. dimensions of
184 the objects appear differently), you might have to change the scale
185 of your meshes in Blender. For this, select all of them, then go to
186 the Object menu and click "Apply->Scale". The same goes for rotation in
187 case there are errors. Also in the Object menu, you can select
188 "Transform->Origin to Geometry", which will fix similar problems with
189 the meshes' positions.
190 
191 However, if you load the scene in MARS and try to do a tank turn you
192 might notice that it does not work to well. You would need to tune
193 the friction parameters for the wheels. Alternatively, we add a third
194 pair of wheels to the model to show the workflow of editing an
195 existing scene.
196 
197 24. Reopen your blender scene if you do not have it opened any more.
198 
199 25. Duplicate a wheel pair (shift+d after selecting the parts) and
200  move it to the middle of the body ("g" for grab; "x" to constrain
201  the movement to the x-axis; "0.25" to move it to the middle)
202 
203  ![New Wheels](images/tutorials/basic_modeling/new_wheels.png)
204 
205 26. Adjust the custom property "node2" of the new joint helper objects
206  to let them point to the new wheels.
207 
208 27. Re-run the "create_mars_props.py" script. You need to execute
209  this script everytime you add or remove an object to/from the
210  scene because it will assign unique ids to all objects and group
211  objects belonging together.
212 
213 28. Run the "relative_mars_export.py" script to update the scene.
214 
215 29. Test in MARS!
