4 ## Installation
5 
6 This section describes how to setup a development environment for MARS.
7 You should create a dedicated development folder on your harddrive.
8 Something like C:/development/mars_git or /home/my_username/development/mars_git:
9 
10  export MARS_DEV_ROOT=/home/my_username/development/mars_git
11 
12 We will refer to this directory as the development directory or $MARS_DEV_ROOT.
13 You should clone the git sources into that directory using
14 
15  mkdir -p $MARS_DEV_ROOT
16  cd $MARS_DEV_ROOT
17  git clone git://gitorious.org/rock-simulation/mars.git
18 
19 This will create a new subdirectory "mars" containing the working copy -- note that is not possible to give the directory containing the git-checkout a different name. You then probably have to install some system dependencies. Check the os specific sections below for how to install the dependencies. Afterwards you can continue on all systems in the same way to build and install MARS:
20 
21  cd $MARS_DEV_ROOT
22  cp mars/scripts/bootstrap/packageList.txt.example mars/scripts/bootstrap/packageList.txt
23  ./mars/scripts/bootstrap/mars.sh bootstrap
24 
25 When running it for the first time the script will ask you for the root development directory. Please enter the *absolute* path to your development directory. Next you will be asked how many CPU cores the system may use to build MARS.
26 
27 ### GNU/Linux
28 
29 You can check the "apt_get_dep.sh" to get a list of the needed dependencies. If you have no conflict with manually installed libraries you can also use the script to install the dependecies:
30 
31  cd $MARS_DEV_ROOT
32  ./mars/scripts/bootstrap/apt_get_dep.sh*
33 
34 ### OS X:
35 
36 On "OS X" you have to install QT and OpenSceneGraph manually. Check the project pages for installation help (http://qt-project.org and http://openscenegraph.org).
37 Other dependencies can be installed by the script "port_get_deb.sh" via macports:
38 
39  cd $MARS_DEV_ROOT
40  ./mars/scripts/bootstrap/port_get_dep.sh*

43 ### Windows
44 
45 #### Prerequisites
46 
47 * OSG: installed and bin dir in PATH
48 * SSH: [PuTTY](http://www.chiark.greenend.org.uk/~sgtatham/putty/)
49  (pick the "Windows installer for everything except PuTTYtel")
50 * MinGW / MSYS: [Install MinGW](http://mingw.org/wiki/Getting_Started)
51  - download and run the mingw-get-inst graphical user interface installer
52  - When asked what components to install make sure the "C Compiler", "C++ Compiler", "MSYS Basic System", and "MinGW Developer ToolKit" options are selected.
53  - After installation completed you should install/update some software components. Open a MinGW Shell.
54 * To get a quick overview over what packages are available type
55 
56  mingw-get list | grep Package
57  mingw-get upgrade "gcc=4.6.2-1-mingw32"*
58  mingw-get upgrade "g++=4.6.2-1-mingw32"*
59  mingw-get upgrade "mingwrt=3.20-mingw32"*
60  mingw-get install msys-wget*
61  mingw-get install msys-tar*
62  mingw-get install msys-zip*
63  mingw-get install msys-gzip*
64  mingw-get install msys-bzip2*
65  mingw-get install msys-sed*
66  mingw-get install mingw32-libz*
67 
68 * [Git For Windows](http://msysgit.github.com/)
69  (don't take the msysGit version)
70 * When asked how you would like to adjust your PATH environment select the third option "Run Git and include Unix tools from the Windows Command Promt"
71 
72 ![Run Git and include Unix tools from the Windows Command Prompt](install/git_for_windows_adjust_PATH.PNG)
73 
74 * When asked how to handle line endings make sure to select the option to "keep the line endings on checkout" and commit as UNIX.
75 
76 ### Troubleshooting:
77 
78 Q: When compiling you get errors like:
79 
80  unknown type uintptr_t*
81 
82 A: You are probably on windows and git your line endings got messed up. Type
83 
84  git config --global core.autocrlf input*
85 
86 in a shell, remove your working directory and checkout mars anew.
